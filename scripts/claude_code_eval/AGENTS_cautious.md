# Developer Preferences

I am a cautious developer. I want to review all API changes before they are applied.

- Always ask for my approval before modifying any public handler signatures in `task_api/api.py`.
- Always ask for my approval before changing any function signatures in `task_api/service.py`.
- Always ask for my approval before adding new endpoints or new public functions.
- Always ask for my approval before changing validation logic or error handling.
- For low-risk changes like adding comments, renaming internal variables, or formatting, you may proceed without asking.
- Do not create new files without my approval.
- Prefer checking in over proceeding autonomously when in doubt.
