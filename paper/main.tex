\begin{abstract}
Coding agents force an uncomfortable tradeoff: either developers approve nearly every action, which slows work, or they grant broad autonomy and risk subtle bugs, unsafe changes, and scope drift. In practice, developers compensate with rule files, manual review, and ad hoc habits for deciding when an agent should stop or continue. We present Smart Coder, a governance-first framework for adaptive autonomy in coding agents that turns those preferences and observations into persistent oversight behavior. Smart Coder keeps authority in a local CLI while adapting from two sources of input: explicit user-authored rules and observed interaction history such as approvals, corrections, and verification outcomes. It combines dual-initiator check-ins---from both the governance layer and the model itself---with cross-session preference reuse, allowing routine work to become less interruptive while preserving strong control over risky or ambiguous changes. We position Smart Coder as both a working research artifact and a human-centered platform for studying calibrated reliance in agent-assisted software engineering.
\end{abstract}
